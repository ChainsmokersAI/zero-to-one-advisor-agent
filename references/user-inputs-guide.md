# User Inputs 시스템 가이드

사용자가 별도로 조사·작성한 원본 자료를 보관하고, 에이전트가 필요 시 참조하기 위한 시스템입니다.

## 개념: 3체계 구분

프로젝트는 3개의 독립된 데이터 체계로 운영됩니다:

- **Memory**: 작업 이력, 학습 기록, 사용자 선호도 등 에이전트의 **경험** 중심
- **Knowledge**: 도메인 용어, 프레임워크, 베스트 프랙티스 등 에이전트가 수집한 **지식** 중심
- **User Inputs**: 사용자가 제공한 고객/시장 조사, 제품/기술 자료 등 **원본 자료** 중심

| 구분 | Memory | Knowledge | User Inputs |
|------|--------|-----------|-------------|
| 정체성 | 에이전트의 경험 기록 | 에이전트가 수집한 도메인 지식 | 사용자가 제공한 원본 자료 |
| 작성 주체 | 에이전트 | 에이전트 | 사용자 |
| 가공 여부 | 에이전트가 정리 | 에이전트가 정리 | 원본 그대로 |
| 참조 트리거 | 세션 시작, skill 실행 전 | 모든 작업 시작 전 | 모든 작업 시작 전 (index.md) |
| 관리 방식 | 오래된 것 탈락 | 오래된 것 재검증 | 사용자 요청 시에만 삭제 |
| 크기 제한 | MEMORY.md 200줄 | 없음 | 없음 |
| 기록 시점 | 작업 완료 후 | 대화 중 즉시 (웹 서칭 직후) | 사용자 자료 제공 시 |

## 디렉토리 구조

```
user-inputs/
├── index.md                    # 자료 목록 테이블 (엔트리포인트)
├── customer-research/          # 고객/시장 조사 자료
└── product-tech/               # 제품/기술 관련 자료
```

## 저장 규칙

1. **원본 보존**: 사용자가 제공한 자료를 그대로 저장합니다. 에이전트가 헤더 추가, 내용 정리, 구조화를 하지 않습니다
2. **크기 제한 없음**: 자료 파일의 크기에 제한이 없습니다
3. **삭제 금지**: 사용자가 명시적으로 삭제를 요청하지 않는 한 자료를 탈락시키거나 재검증하지 않습니다
4. **분류 저장**: 자료 성격에 따라 `customer-research/` 또는 `product-tech/`에 저장합니다

## 에이전트 참조 기준

1. **index.md 확인**: 모든 작업 시작 전, `user-inputs/index.md`를 확인하여 등록된 자료 목록을 파악합니다
2. **관련성 판단**: 현재 작업과 관련된 자료가 있는지 판단합니다
3. **해당 파일 Read**: 관련 자료가 있으면 해당 파일을 Read 도구로 읽어 참조합니다

## 자료 등록 워크플로우

사용자가 자료를 제공하면:

1. **분류 확인**: 고객/시장 조사(`customer-research/`) vs 제품/기술(`product-tech/`) 판단
2. **파일 저장**: 원본 그대로 해당 디렉토리에 저장 (파일명은 내용을 나타내는 kebab-case)
3. **index.md 업데이트**: `user-inputs/index.md`의 자료 목록 테이블에 항목 추가 (자료명, 분류, 등록일, 설명)
