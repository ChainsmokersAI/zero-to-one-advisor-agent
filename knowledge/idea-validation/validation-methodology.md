# 아이디어 검증 방법론

- **최종 업데이트**: 2026-03-16
- **출처 신뢰도**: 저명 인사
- **출처**: Eric Ries (*The Lean Startup*, 2011), Steve Blank (*The Four Steps to the Epiphany*, 2005), Rob Fitzpatrick (*The Mom Test*, 2013), Sean Ellis (PMF Survey)

## 정의

아이디어 검증(Idea Validation)이란, 제품이나 서비스를 본격적으로 구축하기 전에 고객의 문제 가설과 솔루션 가설을 체계적으로 테스트하여, 실제로 고객이 원하고 비용을 지불할 의향이 있는지를 확인하는 과정이다.

## 핵심 프레임워크

### 1. Steve Blank의 Customer Development (4단계)

- 출처: Steve Blank, *The Four Steps to the Epiphany* (2005)
- 핵심 전제: "There are no facts inside your building, so get outside to test them."

| 단계 | 설명 |
|------|------|
| **Customer Discovery** | 고객의 문제를 찾고 테스트. 3가지 축: Problem, Solution, Revenue Model. 30-50명 인터뷰 권장 |
| **Customer Validation** | MVP로 실제 수요 검증, 확장 가능한 세일즈 퍼널 구축, 첫 거래 성사 |
| **Customer Creation** | 마케팅 전략 — 타겟 시장 선정, 포지셔닝, 제품 런칭, 수요 창출 |
| **Company Building** | 스타트업에서 확장 가능한 조직으로 전환 |

(출처: https://gustdebacker.com/customer-development-process/, 리서치 수행일: 2026-03-16)

### 2. Eric Ries의 Build-Measure-Learn Loop

- 출처: Eric Ries, *The Lean Startup* (2011)
- 아이디어를 제품으로 만들고(Build), 고객 반응을 측정하고(Measure), 피봇 또는 유지를 결정하는 학습(Learn)의 반복 루프
- "The fundamental activity of a startup is to turn ideas into products, measure how customers respond, and then learn whether to pivot or persevere."
- MVP 정의: "version of a new product which allows a team to collect the maximum amount of validated learning about customers with the least effort"
- "Remove any feature, process, or effort that does not contribute directly to the learning you seek."

(출처: https://theleanstartup.com/principles, 리서치 수행일: 2026-03-16)

### 3. Validated Learning

- 출처: Eric Ries, *The Lean Startup* (2011)
- "The unit of progress for Lean Startups is validated learning — a rigorous method for demonstrating progress when one is embedded in the soil of extreme uncertainty."
- "Startup success can be engineered by following the process, which means it can be learned, which means it can be taught."

### 4. Pivot or Persevere

- 출처: Eric Ries, *The Lean Startup* (2011)
- 정기적으로 검증 결과를 평가하여 전략을 근본적으로 변경(Pivot)하거나 현재 경로를 유지(Persevere)하는 결정
- Innovation Accounting: "how to measure progress, how to setup milestones, how to prioritize work. This requires a new kind of accounting, specific to startups."

### 5. Sean Ellis의 Product-Market Fit 측정 (40% Rule)

- 출처: Sean Ellis, PMF Survey (2010년경)
- 핵심 질문: "How would you feel if you could no longer use [Product]?"
- 선택지: Very disappointed / Somewhat disappointed / Not disappointed
- **40% 벤치마크**: 40% 이상이 "Very disappointed"라고 답하면 PMF 달성
- 설문 대상: (1) 핵심 경험 고객, (2) 최소 2회 사용, (3) 최근 2주 이내 사용

(출처: https://pmfsurvey.com/, 리서치 수행일: 2026-03-16)

## Smoke Test 유형

| 유형 | 설명 |
|------|------|
| **랜딩 페이지 테스트** | 제품 없이 가치 제안 설명 + 이메일 수집. 냉트래픽 기준 CTA 전환율 >15%면 강한 공명 |
| **데모 영상** | 프로토타입 시연 영상으로 수요 테스트 (Dropbox 사례) |
| **Concierge MVP** | 자동화 없이 수동으로 서비스 제공하며 니즈 파악 |
| **Wizard of Oz MVP** | 사용자에게는 자동화처럼 보이지만 실제로는 수동 운영 |
| **Fake Door Test** | 존재하지 않는 기능 버튼을 배치하여 클릭률로 수요 측정 |

(출처: FasterCapital, Kromatic Real Startup Book, 리서치 수행일: 2026-03-16)

## 실무 워크플로우

1. 문제 가설 수립
2. 고객 인터뷰 (Mom Test)
3. 솔루션 가설 수립
4. Smoke Test (랜딩 페이지/데모 영상)
5. MVP 구축
6. 측정 및 학습 (Innovation Accounting)
7. Pivot or Persevere
8. PMF 달성 (Sean Ellis 40%)
9. 초기 영업 (Founder-Led Sales + Sales Deck)

## 2024-2025 Lean Startup 재해석

- 소프트웨어 산업이 훨씬 포화 — 마케팅과 세일즈의 중요성이 극적으로 상승
- AI가 Build-Measure-Learn 루프를 가속화: No-code 플랫폼으로 Build 가속, AI 분석으로 Measure 가속
- 세 가지 멘탈 시프트:
  1. 실행 속도보다 학습 속도 우선
  2. 내부 의견보다 고객 피드백 우선
  3. 데이터가 증명할 때까지 결정은 되돌릴 수 있다고 취급
