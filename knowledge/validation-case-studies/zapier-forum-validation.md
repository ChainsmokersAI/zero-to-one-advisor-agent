# Zapier 포럼 기반 검증 사례

- **최종 업데이트**: 2026-03-16
- **출처 신뢰도**: 커뮤니티
- **출처**: Indie Hackers, HackerNoon, GrooveHQ

## 개요

Wade Foster와 팀이 Zapier를 포럼 게시와 직접 관찰을 통해 검증한 사례. 고객이 이미 문제를 겪고 있는 곳을 찾아가는 전략의 대표 사례.

## 상세 스토리

### MVP 구축
Startup Weekend Columbia에서 54시간 만에 구축.

### 사전 검증
이벤트 전 여러 사람과 대화하고, 포럼에 게시하여 문제를 검증.

### 포럼 전략
Wade가 제품 지원 포럼에서 통합을 원하는 사용자를 직접 찾아냄. 지원하는 통합에 대한 공개 페이지를 만들고, 파트너들이 자사 사이트에서 링크하도록 함.

### 직접 관찰
초기 사용자에게 접근권을 주고 Skype 통화로 사용 과정을 직접 관찰. 사용자가 어려워하거나 혼란스러워하는 부분을 상세히 메모.

### 전환점
한 초기 고객이 제품이 자동으로 앱 간 데이터를 전송하는 것을 보고 "이게 내 사업을 얼마나 개선할지" 황홀해한 반응 — 이전 제품들의 미온적 반응과 극명한 차이.

## 핵심 교훈

- **이미 문제를 겪고 있는 사용자를 찾아가는 것**이 가장 효과적
- 포럼/커뮤니티에서 "이거 할 수 있나요?" 류의 질문이 수요 신호
- 고객의 실제 사용 과정을 **직접 관찰**하는 것의 가치
- "황홀한 반응"이 PMF의 전조 — 미온적 반응과의 극명한 차이를 체감

## 적용 가이드

- 타겟 고객이 이미 모여있는 포럼/커뮤니티를 찾을 것
- 지원 포럼, Q&A 사이트에서 "이런 기능이 있으면 좋겠다" 류의 글을 검색
- 초기 사용자와 1:1 화상 통화로 사용 과정 직접 관찰
- Paul Graham의 "Do Things That Don't Scale" 체현 사례
