# Buffer 2단계 랜딩 페이지 MVP 사례

- **최종 업데이트**: 2026-03-16
- **출처 신뢰도**: 공식 문서
- **출처**: Buffer 공식 블로그, Joel Gascoigne Medium

## 개요

Joel Gascoigne가 Buffer 아이디어를 2페이지 랜딩 페이지로 검증한 대표적 Smoke Test 사례. 수요뿐 아니라 지불 의향까지 동시에 검증한 영리한 설계로 유명.

## 상세 스토리

### 아이디어
트윗 스케줄링이 수작업으로 너무 오래 걸린다는 개인적 불편함에서 출발.

### 1단계 — 관심도 검증
Buffer의 컨셉을 설명하는 간단한 랜딩 페이지 + "런칭 시 알림 받기" CTA. 트위터에 링크를 공유하고 피드백 요청.

### 2단계 — 지불 의향 검증
원래 2페이지 사이에 **가격 페이지를 삽입**. 방문자가 무료/유료 플랜을 선택한 후에야 이메일 제출 가능.
→ 수요와 가격 관심도를 동시 검증하는 영리한 설계.

### 결과
- 7주 동안 120개 이메일 수집
- 런칭일에 50명 가입
- **3일 만에 첫 유료 고객 확보**
- 런칭일: 2010년 11월 30일, 7주간의 개발 후

## 핵심 교훈

- 실제 제품 없이도 수요뿐 아니라 **지불 의향**까지 검증 가능
- 가격 페이지 삽입이라는 간단한 장치로 '관심'과 '커밋'을 구분
- 전환 퍼널의 각 단계가 검증 데이터를 제공
- 7주라는 짧은 개발 기간으로 첫 유료 고객 달성

## 적용 가이드

- **수요 검증만 필요할 때**: 1단계만 (컨셉 + 이메일 수집)
- **지불 의향까지 검증할 때**: 2단계 모델 (컨셉 → 가격 → 이메일)
- 가격 페이지에서의 이탈률 자체가 지불 의향에 대한 데이터
- CTA는 최대한 구체적으로 ("런칭 시 알림 받기" > "더 알아보기")
