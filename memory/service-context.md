## 고객 문제 가설

AI/LLM 서비스는 사용자에게 높은 자유도를 부여(채팅 UI/UX)하므로, PM 등이 기획한 시나리오(Happy Path) 및 엣지 케이스(Edge Case; Prompt Injection 등 악의적 공격 포함)에 대한 검증이 요구됩니다.

| # | Pain Point | 설명 |
|---|-----------|------|
| 1 | **페르소나 + 시나리오 기획의 어려움** | 다양한 페르소나 + 시나리오를 기획하는 것이 매우 어렵고 소모적인 업무 |
| 2 | **Multi-Turn 시뮬레이션의 소모성** | 페르소나 + 시나리오를 기반으로 Multi-Turn 대화를 이어나가는 것(시뮬레이션)도 매우 소모적인 업무 |
| 3 | **품질 평가의 어려움** | 시뮬레이션 대화 내용을 기반으로 PM, AI Engineer, QA Engineer 등이 문제를 파악/분석하는 것이 어려움 — 통합 평가/분석 환경 부재, 객관적 지표 부재 |

## 솔루션 가설

### 핵심 방향

고객 서비스 정보 수집 → 고품질 페르소나 + 시나리오 생성 → API Endpoint 연동 고객 서비스와 Multi-Turn 대화 시뮬레이션 → 다양한 이해관계자들의 평가 플랫폼 제공

### 우선순위

- **당장 집중**: Pain Point 1, 2 (페르소나/시나리오 생성 + 시뮬레이션)
- **시나리오 구성**: n개의 테스트 케이스로 구성, 각 테스트 케이스 = 페르소나 + 테스트 목적 (예: 욕설 시 존댓말 유지, 악의적 공격 시 보안 정책 준수 등)
- **Pain Point 3 접근**: '품질 평가는 아직 사람이 수동으로 진행해야 한다'는 전제 하에, 다양한 멤버들이 Pass/Fail 또는 Scoring을 할 수 있는 환경 제공 (평가 기준 생성, Rubric 작성 등)
- **LLM-as-a-Judge**: 자동 평가 기능도 제공 예정이나 당장 집중 대상은 아님

## 핵심 포인트

- 서비스 정보 수집을 AI 에이전트(가상 컨설턴트) 대화 방식으로 구현 중
  - 사용자 관점: "최소 정보 입력 → 고품질 결과" (높은 ROI)가 핵심
  - 회사 리더십 관점: "고객이 최대한 많은 정보를 제공하게 만드는 것"에 초점
  - 현재 리더십 방향으로 구현 중 — 대화형 UX로 고객이 재미있게 오래 사용하게끔 설계
- 정교한 페르소나 생성을 매우 중요하게 보는 사내 비전
  - 실리콘밸리에서 가상 페르소나 기반 유저 리서치/시뮬레이션 스타트업들이 대규모 투자 유치 중
  - 미래에 정교한 페르소나 생성이 핵심 역량이 될 것이라는 리더십 비전

## 리스크 및 걱정되는 점

1. 고객이 레퍼런스 없는 제품에 많은 시간을 투자할 것이라는 가정 자체가 위험
2. 고객 인터뷰 결과, 설정한 문제를 명시적으로 겪는 고객이 많지 않았음 — 자체 해결하거나 대수롭지 않게 넘기는 경우 존재
3. 페르소나 생성 자체에 열중하여 QA 문제 해결에 온전히 집중하지 못하는 느낌 — 문제보다 솔루션에 집중하는 경향
