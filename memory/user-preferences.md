# 사용자 선호도

- [2026-03-17] 메모리 시스템 관련:
  - "메모리 시스템이 가장 중요하다고 말해도 손색이 없습니다" — 사용자 직접 발언
  - 실수를 꼼꼼하게 기록하여 절대 재발하지 않기를 원함
  - 작업 후 메모리 전체를 점검(task-log, 실수, 교훈, 선호도)하는 것을 기대
- [2026-03-18] 리서치 작업 관련:
  - 리서치 단계를 나누어 중간 확인을 먼저 요청하는 방식을 선호 (예: 후보 리스트를 먼저 보여주고 사용자 판단 후 전수 조사)
  - 리스크를 사전에 파악하고 대책을 설계하는 접근을 선호 (예: SPA 크롤링 제한 사전 인지)
  - 별도 skill 없이 직접 수행이 적절한 경우, 굳이 skill을 거치지 않는 유연한 판단을 선호
- [2026-03-19] 규칙/시스템 설계 관련:
  - 규칙/시스템 설계 시 확장성(Scalability)을 중시 — 특정 파일명 하드코딩보다 일반화된 패턴 선호
  - 규칙은 자연어로 작성 — glob 패턴(*-context.md) 등 기술적 표현보다 자연스러운 문장 선호
- [2026-03-18] 랜딩 페이지 컨설팅 관련:
  - **구조 제안 시 논의 선호**: "당장 확정 짓기보다는 논의를 통해 정해나가면 좋을 것 같아요" — 초안 제시 → 논의 → 확정 단계를 밟는 것을 선호. 일방적 확정 지양
  - **적극적 콘텐츠 기여자**: 사용자가 직접 카드 내용/방향을 제시함 (Key Capabilities 3개 카드 전면 교체). 에이전트가 일방적으로 쓰는 것이 아니라, 사용자의 아이디어를 받아 다듬는 협업 방식
  - **비즈니스 영어 뉘앙스에 관심**: 영어 표현의 비즈니스 맥락 적절성을 직접 질문함 ("Hyper-Realistic 표현은 좀 별로인가요?"). 카피 제안 시 표현의 비즈니스 영어 적합성을 함께 설명해주는 것이 유용
  - **콘텐츠 중복 민감**: 섹션 간 내용이 겹치면 즉시 지적 ("전 섹션과 겹치는 것 같아서요"). 각 섹션이 고유한 가치를 갖도록 설계해야 함
