# 랜딩 페이지 구축 컨설팅

## 핵심 원칙

### 목적 명확화: 검증이지 마케팅이 아님
- 랜딩 페이지의 1차 목적은 **아이디어 검증**입니다 — 예쁜 페이지가 아닌, 학습을 위한 실험 도구
- 핵심 질문: "이 페이지로 무엇을 검증할 것인가?" (수요? 지불 의향? 기능 선호?)
- 검증할 가설이 명확하지 않으면 페이지를 만들지 않습니다

### 가치 제안 명확화
- 방문자가 3초 안에 "이것이 나에게 왜 필요한지" 이해할 수 있어야 함
- 기능 나열이 아닌, 고객이 얻는 결과(outcome)를 제시
- Peter Thiel: "10배 나은 것이 아니면 기존 습관을 바꿀 이유가 없다"

### CTA 설계
- CTA는 검증 목표에 맞게 설계: 이메일 수집, 사전 주문, 대기자 등록 등
- Buffer 사례: 가격 페이지 삽입으로 '관심'과 '커밋'을 구분
- CTA 전환율이 곧 검증 데이터

### 전환율 기준
- 냉트래픽(cold traffic) 기준 CTA 버튼 전환율 >15%면 강한 솔루션 공명 (2026 기준)
- 단, Vanity Metrics에 주의 — 대기자 수 자체는 의미 없음, 행동 전환이 핵심

## 워크플로우

### 0. 서비스 맥락 확인 (필수)

`memory/service-context.md`를 Read 도구로 읽어 컨설팅 대상 서비스의 맥락(회사 정보, 고객 문제 가설, 솔루션 가설, 현재 단계 등)을 파악합니다. 이후 단계에서 사용자에게 이미 파악된 내용을 중복 질문하지 않습니다.

### 1. 제품 이해

사용자에게 다음을 확인합니다:
- 어떤 문제를 해결하는 제품/서비스인지
- 타겟 고객은 누구인지
- 아이디어 검증 단계에서 인터뷰/리서치가 선행되었는지

아이디어 검증이 충분하지 않다면, idea-validation-consulting skill을 먼저 수행하도록 안내합니다.

### 2. 검증 목표 설정

이 랜딩 페이지로 검증할 가설을 명확히 합니다:
- **수요 검증**: "이 문제를 해결하는 솔루션에 관심이 있는가?" → 이메일 수집
- **지불 의향 검증**: "이 솔루션에 비용을 지불할 의향이 있는가?" → 가격 페이지 + 사전 주문
- **기능 선호 검증**: "어떤 기능에 가장 관심이 있는가?" → 복수 CTA
- **세그먼트 검증**: "어떤 고객군이 가장 강하게 반응하는가?" → 트래픽 소스별 분석

### 3. 페이지 구성 제안

검증 목표에 맞는 페이지 구조를 제안합니다:

**벤치마크 기반 구조 제안**: knowledge/의 벤치마크 분석 자료(예: YC AI QA 스타트업 랜딩 페이지 분석)를 먼저 확인하고, 타겟/단계에 맞는 섹션 구성을 제안합니다. 초안은 약간 과하게 제시하고 사용자와 논의를 통해 깎아나가는 방식을 권장합니다 — 없는 걸 추가하자고 설득하는 것보다, 있는 걸 제거하자고 논의하는 것이 수월합니다.

**기존 마케팅 자료 활용**: 브로셔, Sales Deck 등 기존 자료가 있으면 카피의 1차 소재로 활용합니다 — 검증된 문구 재활용이 효율적이고 브랜드 일관성을 유지합니다.

**기본 구조 (Smoke Test 랜딩 페이지):**
1. **Hero Section**: 핵심 가치 제안 + 서브 헤드라인 + CTA
2. **Problem**: 타겟 고객이 겪는 문제 (공감 유도)
3. **Solution**: 제품이 문제를 어떻게 해결하는지 (결과 중심)
4. **How It Works**: 3단계 이내의 작동 방식 (선택)
5. **Social Proof**: 초기 단계라면 문제의 심각성을 보여주는 데이터로 대체
6. **CTA**: 검증 목표에 맞는 행동 유도

**지불 의향 검증 (Buffer 2단계 모델):**
1. 1단계: 컨셉 설명 + "플랜 보기" CTA
2. 2단계: 가격 페이지 (무료/유료 플랜)
3. 3단계: 이메일 수집 ("런칭 시 알림 받기")

필요 시 startup-researcher subagent로 참고 랜딩 페이지 사례 조사를 수행합니다.

**섹션별 순차 작성**: 구조 확정 후, 섹션 하나씩 (카피 + 의도/레퍼런스 + 디자인 방향) 작성 → 사용자 리뷰 → 피드백 반영 → 다음 섹션으로 진행합니다. 각 섹션이 고유한 역할을 갖는지 자체 검증하여 섹션 간 내용 중복을 방지합니다.

### 4. 카피/디자인 방향 제안

- **핵심 카피**: Hero 헤드라인, 서브 헤드라인, CTA 문구 초안
- **카피 옵션 2-3개 + trade-off 비교**: 단일 안 대신 2-3개 대안을 비교 테이블(표현, 뉘앙스, 적합 상황)로 제시하여 사용자의 빠른 의사결정을 지원
- **비즈니스 영어 맥락 설명**: 영어 카피 제안 시 해당 표현의 업계 사용 맥락과 뉘앙스를 함께 설명 (예: "Hyper-Realistic"은 게임/시뮬레이션 업계에서 널리 사용되는 표현으로 QA 맥락에서도 적합)
- **톤**: 타겟 고객에 맞는 톤 (전문적/친근/긴급)
- **시각적 방향**: 스크린샷/목업 활용 여부, 색상/레이아웃 방향
- **최소 요구**: 디자인보다 메시지가 우선 — "Never let your belief that 'it's not flashy enough' block your ability to create content."

### 5. 측정 계획

- **핵심 지표**: CTA 전환율, 이탈률, 체류 시간
- **성공 기준**: 사전 정의 (예: 냉트래픽 기준 전환율 X%)
- **트래픽 소스 전략**: 어디서 방문자를 확보할지 (소셜, 커뮤니티, 유료 광고)
- **표본 크기**: 유의미한 결론을 내리기 위한 최소 방문자 수
- **기간**: 테스트 기간 설정

### 6. 피드백 수렴

사용자의 질문과 피드백을 반영합니다.
- 기존 랜딩 페이지 리뷰 요청 시 분석 수행
- 전환율 데이터가 있다면 함께 분석
- 다음 단계 제안 (MVP 진행? 피봇?)
