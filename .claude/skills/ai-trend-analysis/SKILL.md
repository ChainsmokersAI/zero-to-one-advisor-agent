# IT/AI 업계 트렌드 분석

## 핵심 원칙

### 데이터 기반 분석
- 주관적 추측이 아닌 데이터와 근거에 기반한 분석만 제공
- 투자 금액, 시장 규모, 성장률 등 정량적 데이터 활용
- 출처와 데이터 시점을 반드시 명시

### 복수 출처 교차 검증
- 단일 출처에 의존하지 않고, 최소 2-3개 출처를 교차 검증
- 상충하는 데이터가 있으면 양쪽을 모두 제시하고 차이 이유 분석
- 출처의 신뢰도를 명시 (공식 보고서 > 업계 분석 > 블로그)

### 최신성 우선
- AI 분야는 6개월이면 판도가 바뀜 — knowledge 데이터의 날짜를 반드시 확인
- 6개월 이상 된 데이터는 "해당 시점 기준"임을 명시
- 최신 데이터가 필요하면 startup-researcher subagent로 실시간 조사

### 실용적 연결
- 트렌드 자체보다 사용자의 아이디어/사업에 대한 시사점에 집중
- Peter Thiel의 7 Questions 프레임워크를 활용하여 전략적 질문 제기
- "지금이 이 사업을 시작할 적기인가?" (The Timing Question) 관점 포함

## 워크플로우

### 0. 서비스 맥락 확인 (필수)

`memory/service-context.md`를 Read 도구로 읽어 컨설팅 대상 서비스의 맥락(회사 정보, 고객 문제 가설, 솔루션 가설, 현재 단계 등)을 파악합니다. 트렌드 분석 시 사용자의 아이디어와 연결할 맥락으로 활용합니다.

### 1. 분석 범위 정의

사용자에게 다음을 확인합니다:
- **산업/기술 영역**: AI 전반? 특정 세그먼트? (예: Vertical AI, Agentic AI)
- **분석 목적**: 시장 진입 가능성? 경쟁 분석? 트렌드 파악? 타이밍 판단?
- **사용자의 아이디어/사업**: 트렌드와 연결할 맥락 (있는 경우)

### 2. 기존 Knowledge 확인

knowledge/ai-startup-trends/ 카테고리에서 관련 자료를 먼저 확인합니다.
- 최신성 확인: 데이터 시점이 6개월 이내인지
- 관련성 확인: 사용자의 분석 범위에 부합하는지
- 부족한 부분 식별

### 3. 리서치 실행

기존 Knowledge가 부족하거나 최신 데이터가 필요한 경우, startup-researcher subagent로 조사합니다:
- 최신 투자 동향 (VC 펀딩, 주요 라운드)
- YC 배치 트렌드
- 경쟁 환경 분석 (주요 플레이어, 차별화 요소)
- 기술 트렌드 (새로운 모델, 인프라, 도구)
- 규제/정책 변화

### 4. 분석 및 인사이트 도출

체계적인 분석 결과를 제공합니다:

**시장 개요:**
- 시장 규모 및 성장률
- 주요 투자 동향
- 핵심 플레이어 맵

**트렌드 분석:**
- 주요 트렌드 3-5개 (근거 + 출처 포함)
- 각 트렌드의 성숙도 (초기/성장/성숙)
- 향후 6-12개월 전망

**사용자 아이디어 연관성** (아이디어가 있는 경우):
- 시장 기회 (Timing Question: 왜 지금인가?)
- 경쟁 리스크 (Monopoly Question: 작은 시장의 큰 점유율로 시작 가능한가?)
- 차별화 기회 (Contrarian Question: 남들이 보지 못하는 진실은?)
- Distribution 전략 시사점

### 5. Knowledge 업데이트

새로운 트렌드 정보가 확인되면 즉시 knowledge/ai-startup-trends/ 카테고리를 업데이트합니다:
- 기존 topic 파일 갱신 또는 새 topic 생성
- 출처, 최종 업데이트 날짜, 신뢰도 표기
- 카테고리 index.md 갱신

### 6. 피드백 수렴

사용자의 질문과 피드백을 반영합니다.
- 특정 영역 심층 분석 요청
- 경쟁사 비교 분석
- 추가 트렌드 탐색
